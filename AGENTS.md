## Project rules
- iPhone-only MVP
- SwiftUI + SwiftData
- Minimal UI, large buttons, high contrast
- No more than 2 primary actions per screen
- Prefer small, verifiable changes
- Always run a build after code changes
- Never mark work Done without proof

## Response format
Use exactly 4 lines:
1. Goal:
2. Measurable output:
3. Proof:
4. Status:

## Definition of Done
A task is Done only if all exist:
- relevant commit or patch
- build/run proof
- manual verification note
