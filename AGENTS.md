# AGENTS.md

## Product
iPhone-only MVP for a hands-free grocery shopping app.

## Tech stack
- SwiftUI
- SwiftData
- AVSpeechSynthesizer
- iPhone only
- Local-only storage for MVP

## UX rules
- Large, one-hand-friendly controls
- High contrast in light/dark mode
- Immediate visual and spoken feedback
- No more than 2 primary actions per screen
- Consistent spacing and alignment
- Accessibility-friendly text sizing where practical

## Delivery rules
- Prefer small, verifiable changes
- Modify code directly instead of creating planning markdown files
- Keep pull requests small and focused
- One task per branch
- One pull request per task
- Do not merge automatically
- Run a build after every meaningful code change
- Add lightweight debug logs for important user actions

## Definition of Done
A task is Done only if all are present:
1. Relevant code changes
2. Build/run proof
3. Manual verification note

If any of the above is missing, mark the task as Doing or Blocked.

## Required final response format
Use exactly 4 lines:

1. Goal:
2. Measurable output:
3. Proof:
4. Status:

Keep it short, concrete, and verifiable.
